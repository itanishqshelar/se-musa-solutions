# â­ **2. Prototype Model (Detailed Explanation)**

The **Prototype Model** is an SDLC model where a **working model of the software** is built early to understand requirements, especially when requirements are **unclear or incomplete**.

---

# ğŸ”¹ **Definition (Exam-Ready)**

**The Prototype Model is a software development model in which a prototype (temporary working model) is built, tested, and refined based on user feedback until the exact requirements are understood.**

---

# ğŸ”¹ **Phases of Prototype Model**

### **1. Requirements Gathering**

- Only basic & incomplete requirements are collected.
- Focus on unclear and ambiguous parts.

---

### **2. Quick Design**

- A rough design is prepared.
- Concentrates on important screens, input/output, main functions.

---

### **3. Build Prototype**

- A simple, working prototype is created.
- May include simple UI, limited functionality.

---

### **4. User Evaluation**

- Users interact with the prototype.
- Give feedback on missing features, improvements, changes.

---

### **5. Refinement**

- Based on feedback:
  âœ“ Fix issues
  âœ“ Add new features
  âœ“ Improve design
- Prototype is updated and shown again.

---

### **6. Final System Development**

- Once requirements are clear, **actual system** is developed using suitable SDLC model.

---

### **7. Testing and Maintenance**

- Final product is tested and then maintained.
